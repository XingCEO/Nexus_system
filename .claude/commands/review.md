# 程式碼審查

對當前變更進行 AI 程式碼審查，找出潛在問題並提供改善建議。

## 執行步驟

1. 取得當前 Git 變更（staged + unstaged）
2. 分析程式碼變更
3. 檢查以下項目：
   - 邏輯錯誤
   - 安全漏洞（OWASP Top 10）
   - 效能問題
   - 程式碼風格一致性
   - 型別安全
   - 測試覆蓋
4. 輸出審查報告

## 審查等級

- `quick` - 快速檢查（僅重大問題）
- `standard` - 標準審查（預設）
- `thorough` - 深度審查（含架構建議）

## 輸出格式

```
## 審查摘要
- 🔴 嚴重問題：X 項
- 🟡 建議改善：X 項
- 🟢 良好實踐：X 項

## 詳細報告
...
```

## 範例

```
/review
/review --level thorough
/review --staged-only
```
