# 無障礙檢查

檢查和改善應用程式的無障礙性（Accessibility / a11y）。

## 標準

- **WCAG 2.1 AA** - 基本要求
- **WCAG 2.1 AAA** - 進階要求（可選）

## 執行步驟

1. 執行自動化檢查（axe-core）
2. 分析檢查結果
3. 按嚴重程度分類
4. 提供修復建議
5. 生成報告

## 檢查項目

| 類別 | 項目 |
|------|------|
| **感知** | 替代文字、色彩對比、字體大小 |
| **操作** | 鍵盤導航、焦點管理、觸控目標 |
| **理解** | 標籤、錯誤訊息、一致性 |
| **穩固** | 語意 HTML、ARIA 屬性 |

## 嚴重程度

- 🔴 **Critical** - 必須立即修復
- 🟠 **Serious** - 應該修復
- 🟡 **Moderate** - 建議修復
- 🔵 **Minor** - 可選改善

## 常見問題

```
❌ 圖片缺少 alt 屬性
❌ 按鈕只有圖標沒有 aria-label
❌ 表單輸入缺少 label
❌ 色彩對比度不足（< 4.5:1）
❌ 焦點樣式被移除
❌ 使用非語意元素
```

## 指令選項

- `--check <path>` - 檢查特定頁面
- `--level <AA|AAA>` - 指定標準等級
- `--fix` - 自動修復簡單問題
- `--report` - 生成完整報告

## 範例

```
/a11y
/a11y --check src/app/chat/page.tsx
/a11y --level AAA
/a11y --fix
```
